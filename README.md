# 신라면세점 모바일 상품/혜택 조회 웹앱

상품명, SKU, 신라 상품 URL을 입력하면 신라면세점 모바일 검색 결과 페이지를 Playwright로 열고, 결과 카드에서 상품명, 브랜드, 상품 URL을 추출해 표로 보여줍니다.

엑셀 또는 CSV 파일을 올려 여러 검색어를 한 번에 조회할 수 있습니다. 화면의 `엑셀 양식 다운로드` 버튼으로 양식을 받은 뒤 `입력값` 열에 브랜드명, SKU, 상품 URL을 한 줄에 하나씩 넣으면 됩니다. 기존 파일을 올릴 때는 첫 번째 시트에서 `입력값`, `검색어`, `SKU`, `상품명`, `상품 URL`, `브랜드명` 열을 우선 읽고, 헤더가 없으면 첫 번째 열을 검색어로 사용합니다. `검색결과 일괄 조회`는 검색 결과 목록에서 보이는 값만 추출하고, `로그인 혜택 조회`는 상품 상세페이지까지 들어가 혜택 정보를 확인합니다.

로그인 세션이 등록되어 있으면 `로그인 혜택 조회` 버튼은 상품 상세페이지까지 들어가 로그인 기준 `신라할인율`, `신라적립율`, `신라S포인트`를 함께 추출합니다. 로그인 세션이 없으면 비회원 기준으로 조회합니다. 상단 검색과 `검색결과 일괄 조회`는 검색 결과 목록 조회라서 로그인 세션이 있어도 상세 혜택 기준으로 계산하지 않습니다.

## 실행

```bash
npm install
npm run install:browsers
npm start
```

브라우저에서 `http://localhost:3000`을 엽니다.

Playwright Chromium이 아직 설치되지 않은 경우 서버는 설치된 Google Chrome도 자동으로 시도합니다. 직접 지정하려면 `CHROME_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" npm start`처럼 실행할 수 있습니다.

## 로그인 혜택 조회

웹앱이 신라면세점 로그인 혜택을 보려면 서버에 로그인 세션이 등록되어 있어야 합니다. 사용자의 브라우저에서 신라면세점에 로그인했다고 해서 Render 서버가 그 쿠키를 자동으로 읽을 수는 없습니다.

로컬 Mac에서 세션을 만드는 방법:

```bash
npm run capture:login
```

열린 Chrome 창에서 신라면세점에 로그인한 뒤 터미널에서 Enter를 누르면 아래 파일이 생성됩니다.

- `.shilla-storage-state.json`: 로컬 실행용 로그인 세션
- `.shilla-storage-state.base64.txt`: Render 환경변수에 붙여넣을 값

Render에서 회원 혜택 조회를 쓰려면 `Environment`에 다음 값을 추가합니다.

- `SHILLA_STORAGE_STATE_BASE64`: `.shilla-storage-state.base64.txt` 파일 내용 전체
- `SHILLA_LOGIN_LABEL`: 결과 파일에 표시할 계정 구분명. 예: `내 신라계정`

세션 파일은 로그인 쿠키를 포함하므로 GitHub에 올리면 안 됩니다. 이 저장소의 `.gitignore`와 `.dockerignore`는 해당 파일을 제외합니다.

## 공개 URL로 배포

아이폰이나 외부 기기에서 `localhost`로 접속할 수 없습니다. `localhost`는 각 기기 자기 자신을 뜻하므로, 어디서든 접속하려면 Render, Fly.io, Railway, VPS 같은 서버 실행형 호스팅에 배포해야 합니다.

이 앱은 Playwright로 Chromium을 실행하므로 정적 사이트 호스팅이 아니라 Docker 기반 웹 서비스로 배포하는 편이 안전합니다. 이 저장소에는 `Dockerfile`이 포함되어 있습니다.

Render 예시:

1. 이 폴더를 GitHub 저장소로 올립니다.
2. Render에서 `New > Web Service`를 만들고 GitHub 저장소를 연결합니다.
3. Runtime 또는 Language는 `Docker`를 선택합니다.
4. Dockerfile 경로는 저장소 루트의 `Dockerfile`을 사용합니다.
5. 배포가 끝나면 `https://...onrender.com` 형태의 공개 URL이 생깁니다.

헬스체크 URL은 `/healthz`입니다.

## 동작 범위

- 로그인 세션이 없으면 비회원 조회로 동작합니다.
- 로그인 세션이 있으면 `로그인 혜택 조회`에서 로그인 기준 상세 혜택을 추출합니다.
- 검색 URL은 한국어 모바일 웹 기준 `https://m.shilladfs.com/estore/kr/ko/search?text=...`를 사용합니다.
- 일부 브랜드는 신라 검색이 영문 브랜드명에 더 잘 반응하므로 `헤라`, `헤네시`, `조니워커`, `오쏘몰` 같은 입력값은 영문 검색어로 자동 보정합니다.
- 신라 상품 URL(`/estore/kr/ko/p/숫자`)을 입력하면 `/p/` 뒤 상품코드를 추출해 검색합니다.
- 검색결과 일괄 조회는 입력값당 최대 10건씩 추출하며, 한 파일에서 최대 200개 입력값까지 처리합니다.
- 로그인 혜택 조회는 입력값당 첫 번째 매칭 상품 상세페이지를 확인합니다.
- SKU 앞 4자리는 `브랜드코드`로 자동 추출합니다. 같은 결과 안에서 브랜드코드가 확인된 브랜드는 SKU가 비어 있는 행에도 같은 브랜드코드를 표시합니다.
- 기본 추출 건수는 50건이며 서버에서 최대 100건으로 제한합니다.

## 참고
