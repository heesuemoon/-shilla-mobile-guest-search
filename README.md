# 면세점 3사 상품/혜택 조회 웹앱

브랜드명, SKU, 상품 URL을 입력하면 신라면세점, 롯데면세점, 신세계면세점 결과를 Playwright로 열고 상품별 3사 비교표로 보여줍니다.

엑셀 또는 CSV 파일을 올려 여러 검색어를 한 번에 조회할 수 있습니다. 화면의 `엑셀 양식 다운로드` 버튼으로 양식을 받은 뒤 `입력값` 열에 브랜드명, SKU, 상품 URL을 한 줄에 하나씩 넣으면 됩니다. 기존 파일을 올릴 때는 첫 번째 시트에서 `입력값`, `검색어`, `SKU`, `상품명`, `상품 URL`, `브랜드명` 열을 우선 읽고, 헤더가 없으면 첫 번째 열을 검색어로 사용합니다. 파일을 선택하면 화면에 파일명, 용량, 읽은 입력값 개수가 표시됩니다.

상단 `검색`과 `파일 일괄 조회`는 같은 3사 통합 조회를 실행합니다. 세 사이트 모두 앱 전용 Chrome 프로필의 로그인 상태로 상세페이지와 혜택 영역을 읽습니다. 로그인 결과가 확인되지 않으면 비회원가를 표에 섞지 않고 조회를 중단합니다. 브랜드명은 한글/영문 후보를 함께 조회한 뒤 중복 상품과 브랜드 불일치 상품을 제거합니다.

엑셀 결과는 다음 고정 컬럼으로 저장됩니다.

`입력값`, `브랜드`, `상품명`, `신라 SKU`, `신라 정가$`, `신라 상품할인%`, `신라 데일리%`, `신라 이벤트(원)`, `신라 최혜$`, `신라 최혜(원)`, `신세계 할인%`, `신세계 적립금가`, `신세계 최혜$`, `신세계 최혜(원)`, `롯데 할인%`, `롯데 적용가능 최대혜택($)`, `롯데 최혜$`, `롯데 최혜(원)`, `최저 최혜(면세점)`, `신라 상품 URL`

가격과 할인율은 헤더에 단위를 붙이고 셀에는 숫자만 넣습니다. `신라 SKU`는 12자리 SKU가 확인될 때만 채웁니다.

## 실행

```bash
npm install
npm run install:browsers
npm start
```

브라우저에서 `http://localhost:3000`을 엽니다.

Playwright Chromium이 아직 설치되지 않은 경우 서버는 설치된 Google Chrome도 자동으로 시도합니다. 직접 지정하려면 `CHROME_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" npm start`처럼 실행할 수 있습니다.

## 로그인 혜택 조회

웹앱이 로그인 혜택을 보려면 서버가 쓰는 Chrome 프로필에 로그인 세션이 있어야 합니다. 사용자의 일반 Chrome에서 면세점에 로그인했다고 해서 Render 서버가 그 쿠키를 자동으로 읽을 수는 없습니다.

로컬 Mac에서 세션을 만드는 방법:

```bash
npm run capture:login
```

열린 Chrome 창은 앱 전용 Chrome입니다. 평소 쓰는 Chrome에 로그인되어 있어도 이 앱 전용 Chrome에서는 다시 로그인해야 합니다. 열린 Chrome 창에서 신라면세점, 롯데면세점, 신세계면세점에 각각 로그인한 뒤 터미널에서 Enter를 누릅니다. 이어서 스크립트가 신라 상품 상세페이지를 열면, 상세 화면이 정상으로 보이는 것까지 확인한 뒤 다시 Enter를 누릅니다. 그러면 아래 파일이 생성됩니다.

- `.shilla-storage-state.json`: 로컬 실행용 로그인 세션
- `.shilla-storage-state.base64.txt`: Render 환경변수에 붙여넣을 값

Render에서 회원 혜택 조회를 쓰려면 `Environment`에 다음 값을 추가합니다.

- `SHILLA_STORAGE_STATE_BASE64`: `.shilla-storage-state.base64.txt` 파일 내용 전체
- `SHILLA_LOGIN_LABEL`: 결과 파일에 표시할 계정 구분명. 예: `내 신라계정`

세션 파일은 로그인 쿠키를 포함하므로 GitHub에 올리면 안 됩니다. 이 저장소의 `.gitignore`와 `.dockerignore`는 해당 파일을 제외합니다.

주의: 면세점 보안 확인이 Render 서버 접속을 막으면 로그인 쿠키가 있어도 상세 혜택 조회가 실행되지 않습니다. Render는 내 노트북이 아니라 밖에 있는 다른 컴퓨터라서 사이트가 막을 수 있습니다. 이 경우 로그인 혜택 조회는 Mac에서 `npm start`로 로컬 실행해야 합니다. 회사 컴퓨터에서 쓰려면 회사 컴퓨터에서도 `npm run capture:login`으로 세션을 다시 만들어야 합니다.

## 공개 URL로 배포

아이폰이나 외부 기기에서 `localhost`로 접속할 수 없습니다. `localhost`는 각 기기 자기 자신을 뜻하므로, 어디서든 접속하려면 Render, Fly.io, Railway, VPS 같은 서버 실행형 호스팅에 배포해야 합니다.

이 앱은 Playwright로 Chromium을 실행하므로 정적 사이트 호스팅이 아니라 Docker 기반 웹 서비스로 배포하는 편이 안전합니다. 이 저장소에는 `Dockerfile`이 포함되어 있습니다.

Render 예시:

1. 이 폴더를 GitHub 저장소로 올립니다.
2. Render에서 `New > Web Service`를 만들고 GitHub 저장소를 연결합니다.
3. Runtime 또는 Language는 `Docker`를 선택합니다.
4. Dockerfile 경로는 저장소 루트의 `Dockerfile`을 사용합니다.
5. 배포가 끝나면 `https://...onrender.com` 형태의 공개 URL이 생깁니다.

헬스체크 URL은 `/healthz`입니다.

## 동작 범위

- 3사 통합 조회는 신라, 롯데, 신세계를 순서대로 확인합니다.
- 로그인 상세 결과가 확인되지 않는 사이트의 비회원/검색목록 값은 결과표에 섞지 않습니다.
- 쿠폰 다운로드는 사이트가 로그인이나 본인 확인을 요구하면 `로그인 필요` 또는 실패 사유만 기록합니다.
- 롯데 검색 URL은 `https://kor.lottedfs.com/kr/search?comSearchWord=...`를 사용합니다.
- 신세계 검색 URL은 `https://www.ssgdfs.com/kr/search/resultsTotal?query=...`를 사용합니다.
- 일부 브랜드는 사이트 검색이 영문 브랜드명에 더 잘 반응하므로 `헤라`, `헤네시`, `조니워커`, `오쏘몰` 같은 입력값은 영문 검색어도 함께 시도합니다.
- 12자리 SKU는 SKU로만 사용합니다. 브랜드코드는 12자리 SKU의 앞 4자리만 사용하고, SKU가 확인되지 않으면 빈칸으로 둡니다.
- 기본 추출 건수는 100건이며 서버에서 최대 100건으로 제한합니다.

## 참고

각 면세점 페이지 구조가 바뀌면 `src/server.js`의 카드 선택자와 가격 선택자를 조정해야 합니다.
